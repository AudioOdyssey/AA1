Welcome to our Community of Contributors!

AudioOdyssey, Inc. contributors are a community of passionate individuals who love creating stories that are fun and also encourage safe driving habits! This website is where you will be able to create stories that will be available for purchase by our users on our app (AudioOdyssey). This help page provides a description of all of the information fields you will be offered while creating your story. Please note that your progress is saved automatically each time you edit any form. Also note that before your story is published it will be verified by an employee of AudioOdyssey, Inc. Overly offensive content will not be published. We hope you find this page helpful and feel free to use the contact page to ask us any additional questions!


Main Story Entry Form

This important form is where you will be routed to automatically when you begin a new story. It contains general information about your story and is also a place where you can view lists of all the locations, objects and events you have entered so far. You can click on the boxes containing information about locations, objects or events to edit them. To add new objects, events or locations, select the appropriate add button on the right hand side.


Fields you may fill out for each Story are:

Story Title: Each story requires a title

Synopsis: Enter a brief summary of your story here

Genre: Select a genre for your story from the drop down menu

Price: Enter a suggested price for your story (USD)

Length: Enter the number of hours you expect a user to go through this story

Inventory Size: Enter the number of objects a user can hold at a given time.

Starting Location: Select the location your story starts on. Note that no locations will appear as options until you create them below.

Cover Photo: This will photo should represent your story and will appear to users on the app



Individual Entry Pages:

Individual entry pages can be accessed by clicking the add buttons on the story page or by selecting existing objects, events or locations to edit them. In addition to information about the location, object or event, all individual entry page include buttons to navigate around your story, a save button and a delete button.

Location Entry

Locations contain the main audio messages that are played each time the user enters a new "location" in the game. Locations do not necessarily need to be literal locations as you would think of them in real life (ex "the grocery store" "school" etc.). They can also be abstract "locations" that represent points in the your story (anywhere a new description is required). You can switch to treeview to view your locations and decisions in a tree.


Fields you may fill out for each Location are:

Location Name: Each location requries a name (will not be seen by user)

Original Description: This will be read the first time a player goes to this location

Short Description: This will be read any subsequent time a player enters this location (can be the same as above if desired)
Select Event Name: Select the name of the event which causes the post event description to be read, or use the link to navigate to the create event page

Post Event Description: This will be read when the user enters this location after it has been affected by an event

Use the blue "edit decision" link at the bottom of the page to add decisions.


Decision Entry

Decisions are the options which users are presented with everytime they enter a new location. Each location can have as many of as few decisions as you would like. The decision entry page can be accessed throught the blue "edit decision" link at the bottom of the location entry page but will only be available while auto go to is unchecked.


Fields you may fill out for each Decision are:

Decision Name: Each decision requires a name (will not be seen by user)

Decision Sequence Number: This number must be between 1 and the number of decisions for thsi location. It represents the order in which the decisions are read (first decision has sequence number 1, second decision has 2 etc.)

Decision Description: This is read each time this decision is presented to the user

Causes a Transition: Check this box if this decision causes a transition to a new location

Select Transition Location: Only can be filled out if transition is checked. Select name of the location this decision transitions into or use the link to navigate to the create location page

Can only Occur Once: Check this box if you only want the user to be presented with this decision once

Is Hidden? Check this box if you don't want the user to be presented with this decision until it has been unhidden by an event or object

Select Event/Object to Show Decision: Select an object OR an event which will unhide the decision. Or you may use the links to navigate to the create event or object page

Is Locked? Check this box if you want this decision to be unavailable but still give a description of it to the user

Select Object/Event to Unlock Decision: Select an object OR an event which will unlock this decision. Or you may use the links to navigate to the create event or object page

Description when Locked: This despcription is read while the decision is locked and will most likely be a hint about how to unlock this description

Description of Decision Aftermath: This optional description will be read after the decision is selected by the user

Does this Decision Trigger an Event? Check this box if this decision causes an event to occur

Select Event Triggered By This Decision: Only able to be filled in if trigger event is checked. Select the name of the event that is caused by this decision or use the link to navigate to the create event page
Select Event to lock Decision: Select an event which will lock this decision. Or you may use the link to navigate to the create new event page

Description when Locked: This despcription is read when the decision is initially locked and will most likely be a hint about how to unlock this description



Object Entry


Objects are anything that can be picked up and moved throughout locations in a story. Note: things that can NOT be transferred between locations do not need to be entered as objects and can just be written about in a description. Also note that items entered as objects do not need to be mentioned in a description because upon entry to a location the user will be told which objects are in the room (unless the object is hidden).


Fields you may fill out for each Object are:

Object Name: Each object requires a name

Select Starting Location: Select the name of the location you want this object to start the game in or use the link to create a new location. Remember objects can be moved from location to location

Description: Enter a brief description of this object that will be read when a user encounters this object

Is Hidden? Check this box if you don't want the user to be able to see this object until a specific event occurs

Select Unhide Event Name: Only will be able to be filled in if the above is checked. Select the name of the event that makes this object available or use the link to create a new event.



Event Entry


Events are things that change the state of a location or of the game. Occurences only need to be entered as events if they will affect decisions later, otherwise they can just be entered as part of a decsription. Events can be triggered by either a decision or location (or both if desired)


Fields you may fill out for each Event are:

Event Name: Each event requires a name

Select Location: Select the location in which this event occurs or use the link to create a new location. This can be left blank if this event is only truggered by a decision.

Description: This description will be read when this event occurs

Is global: Check this box if this event will effect more than the location you selected above


Verification and Publication Information


All stories must be verified before they are available for purchase. Any aspect of your story which is not verified will be returned to you with comments and can be viewed by selecting the view verification information button located on the bottom of the story page. Additionally, verification information will appear on your dashboard.Only submit a story for verification once it is complete. Once your story has been verified it will show up in your "verified" stories. Here you can submit it for publication to the app and your story will then be available for purchase on the app. Once it is published it will show up in your published stories.


Thank You for your interest! We look forward to experiencing your AudioOdyssey!
